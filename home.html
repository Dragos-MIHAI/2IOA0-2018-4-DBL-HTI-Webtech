Welcome to VizYou

Visualizing data: a method that can find order in chaotic material and which can give us a lot of information that doesn’t directly follow from the data itself. It’s easy to show people data rather than explain it to them, which is both more convincing and easier for the human brain to recall. Or it can be used to quickly find data without a lot of information, by grouping data that share the same properties. On the other hand, though, it is a method that can convince people of a false truth by showing part of the data. People can be manipulated without even lying about the data, by means of the correct visualization. It is this exact reason that, while visualization is super important, visualizing data can have many risks.

What we aim to achieve is an easy way to visualize your data efficiently, by uploading a .csv file as data. Visualizing data can be a lot of work when you don’t fully understand the data itself, but this website will provide a quick way to visualize it for you to gain a better understanding of the data itself. To prevent false conclusion from the visualization, more than one visualization is provided which can be compared. In addition, they are all interactive visualizations, to provide the user with an exploratory analysis of the data.

To get started, navigate to the tab ‘Visualization’ and start uploading your big data!

Our Work

This website has been created in order to develop an interactive web-based visualization tool for static graphs/networks. In this website a tool was build and design in such a way that it contains at least two views: one for a node-link diagram and one for an adjacency matrix. Both views should be linked interactively, for example if someone changes one view, the other one should adapt to this change accordingly. From an algorithmic perspective, the node-link diagram is supporting at least 3 different advanced layout approaches (radial, force-directed, hierarchical) while the adjacency matrix is having at least 5 different matrix reordering techniques (Behrisch et al.). Moreover, further interaction techniques have been integrated following the survey by Yi et al.

Force-Directed Node-Link Diagram

- all the edges are of more or less equal length

- few crossing edges as possible

- require no special knowledge about graph theory such as planarity

Hierarchical Node-Link Diagram

- Hierarchies are usually represented as tree visual structures

- Trees tend to be easier to lay out and interpret than networks (e.g. no cycles)

Radial Node-Link Diagram

- expands outwards, radially

- first level is a special case because all the nodes have the same parent.

- the nodes for level 1 can be distributed evenly, or weighted depending on the number of children they have
