Our Work

This website has been created in order to develop an interactive web-based visualization tool for static graphs/networks. In this website a tool was build and design in such a way that it contains at least two views: one for a node-link diagram and one for an adjacency matrix. Both views should be linked interactively, for example if someone changes one view, the other one should adapt to this change accordingly. From an algorithmic perspective, the node-link diagram is supporting at least 3 different advanced layout approaches (radial, force-directed, hierarchical) while the adjacency matrix is having at least 5 different matrix reordering techniques (Behrisch et al.). Moreover, further interaction techniques have been integrated following the survey by Yi et al.

Force-Directed Node-Link Diagram

- all the edges are of more or less equal length

- few crossing edges as possible

- require no special knowledge about graph theory such as planarity

Hierarchical Node-Link Diagram

- Hierarchies are usually represented as tree visual structures

- Trees tend to be easier to lay out and interpret than networks (e.g. no cycles)

Radial Node-Link Diagram

- expands outwards, radially

- first level is a special case because all the nodes have the same parent.

- the nodes for level 1 can be distributed evenly, or weighted depending on the number of children they have
